# nvocc-mailer 部署文档

## 一、系统概述

nvocc-mailer 是航交所内部批量邮件发送系统，用于向 NVOCC 公司发送运价备案相关通知邮件。

**主要功能：**
- 上传 Excel/CSV 名单批量发送邮件
- 富文本编辑邮件正文（支持加粗、颜色、中文字号）
- 草稿保存与加载
- 发送进度实时显示
- 发送中断保护与断点续传
- 主动暂停与恢复
- 重试失败邮件
- 退信检测（POP3）与退信名单下载

---

## 二、技术架构

| 组件 | 技术 |
|---|---|
| 后端 | Python 3.12 + Flask 3.1 + Gunicorn |
| 数据库 | MySQL 8.0 |
| 前端 | Bootstrap 5 + Quill.js |
| 容器化 | Docker |
| 邮件发送 | SMTP（企业邮件服务器） |
| 退信检测 | POP3（企业邮件服务器） |

**容器构成：**
- `nvocc-mailer-app`：Flask 应用，对外暴露 5001 端口
- `nvocc-mailer-db`：MySQL 数据库，仅内部网络访问

---

## 三、部署前提条件

- 目标服务器已安装 Docker（版本 20.10 以上）
- 服务器可访问企业邮件服务器（当前为 192.168.32.8）
- 本地已安装 Docker Desktop 和 Git

## 四、首次部署

### 4.1 获取代码

```bash
git clone https://github.com/22570100410/nvocc-mailer.git
cd nvocc-mailer
```

### 4.2 构建镜像

```bash
docker build -t nvocc-mailer:1.0.0 .
```

### 4.3 打包镜像

```bash
docker save nvocc-mailer:1.0.0 | gzip > nvocc-mailer.tar.gz
```

### 4.4 传输到服务器

```bash
scp nvocc-mailer.tar.gz root@192.168.1.74:/root/
```

### 4.5 在服务器上创建网络和数据卷

SSH 进入服务器后执行：

```bash
docker network create nvocc-mailer_mailer-net
docker volume create nvocc-mailer_db_data
docker volume create nvocc-mailer_uploads
```

### 4.6 启动 MySQL 容器

```bash
docker run -d \
  --name nvocc-mailer-db \
  --restart always \
  --network nvocc-mailer_mailer-net \
  -e MYSQL_ROOT_PASSWORD=Miluo@123 \
  -e MYSQL_DATABASE=nvocc_mailer \
  -v nvocc-mailer_db_data:/var/lib/mysql \
  --command --default-authentication-plugin=mysql_native_password \
  mysql:8.0
```

等待约 30 秒让 MySQL 完全启动。

### 4.7 初始化数据库表结构

```bash
docker exec -i nvocc-mailer-db mysql -uroot -pMiluo@123 nvocc_mailer < init.sql
```

### 4.8 加载镜像并启动应用容器

```bash
docker load < /root/nvocc-mailer.tar.gz

docker run -d \
  --name nvocc-mailer-app \
  --restart always \
  -p 5001:5000 \
  --network nvocc-mailer_mailer-net \
  -v nvocc-mailer_uploads:/app/uploads \
  -e TZ=Asia/Shanghai \
  -e DB_HOST=db \
  -e DB_PORT=3306 \
  -e DB_NAME=nvocc_mailer \
  -e DB_USER=root \
  -e DB_PASSWORD=Miluo@123 \
  -e LOGIN_USERNAME=admin \
  -e LOGIN_PASSWORD=Miluo@2026 \
  -e SECRET_KEY=nvocc-mailer-2026-secret-key \
  -e SMTP_HOST=192.168.32.8 \
  -e SMTP_PORT=25 \
  -e SMTP_USER=filingmail@message.sse.net.cn \
  -e SMTP_PASSWORD=Mot@SsE-65151166+zZ \
  -e SMTP_FROM_NAME=航交所 \
  -e SMTP_USE_SSL=false \
  -e POP3_HOST=192.168.32.8 \
  -e POP3_PORT=110 \
  -e POP3_USE_SSL=false \
  nvocc-mailer:1.0.0
```

### 4.9 验证部署

```bash
docker ps | grep nvocc
docker logs nvocc-mailer-app --tail 10
```

日志中出现 `Listening at: http://0.0.0.0:5000` 说明启动成功。

浏览器访问 `http://192.168.1.74:5001`，使用账号 `admin / Miluo@2026` 登录。

## 五、更新部署

### 5.1 代码有改动（需重新打包）

在本地项目目录执行：

```bash
# 1. 构建新镜像
docker build -t nvocc-mailer:1.0.0 .

# 2. 打包
docker save nvocc-mailer:1.0.0 | gzip > nvocc-mailer.tar.gz

# 3. 传到服务器
scp nvocc-mailer.tar.gz root@192.168.1.74:/root/
```

SSH 进入服务器执行：

```bash
# 4. 加载新镜像
docker load < /root/nvocc-mailer.tar.gz

# 5. 重建应用容器（数据库不动）
docker stop nvocc-mailer-app && docker rm nvocc-mailer-app

# 6. 用新镜像启动（命令同 4.8）
docker run -d --name nvocc-mailer-app ...
```

### 5.2 只改了 Python 文件（不需要重新打包）

适用于只修改了 `app.py`、`config.py` 等 Python 文件的情况，速度更快：

```bash
# 本地传文件到服务器
scp app.py root@192.168.1.74:/tmp/app.py

# 服务器上复制进容器并重启
docker cp /tmp/app.py nvocc-mailer-app:/app/app.py
docker restart nvocc-mailer-app
```

### 5.3 只改配置（换邮箱账号等）

不需要重新打包，直接重建容器修改环境变量即可：

```bash
docker stop nvocc-mailer-app && docker rm nvocc-mailer-app
# 用新的 -e 参数重新 docker run（命令同 4.8，修改对应环境变量）
```

## 八、邮件服务器架构说明

```
我们的系统
    ↓ SMTP 发件（端口 25）
192.168.32.8（邮件网关）
    ↓ 内部转发
180.168.32.10（内部邮件服务器）
    ↓ 发送给各 NVOCC 公司
```

- **发件**：通过 192.168.32.8:25 发送，无 SSL
- **退信检测**：通过 192.168.32.8:110 (POP3) 读取 filingmail 收件箱
- 退信通知发件人为 `MAILER-DAEMON@mail.message.sse.net.cn`，标题为 `Undelivered Mail Returned to Sender`

---

## 九、已知限制

- 退信检测只能检测到有退信通知回到 `filingmail` 收件箱的情况
- 用户账号不存在（域名有效但账号无效）的退信，取决于对方服务器是否发回通知
- 系统为串行发送，1万封邮件预计需要 30-50 分钟（内网环境）

## 十、故障排查

**问题：页面打不开**
```bash
docker ps | grep nvocc-mailer-app   # 确认容器在运行
docker logs nvocc-mailer-app --tail 20  # 查看错误日志
```

**问题：发邮件失败**
- 确认 SMTP 服务器地址和端口正确
- 确认账号密码正确
- 确认 192.168.1.74 能访问 192.168.32.8：`ping 192.168.32.8`

**问题：退信检测失败**
- 确认 POP3 配置正确
- 手动测试连接：
```bash
docker exec nvocc-mailer-app python3 -c "
import poplib
m = poplib.POP3('192.168.32.8', 110)
m.user('filingmail@message.sse.net.cn')
m.pass_('Mot@SsE-65151166+zZ')
print('连接成功，邮件数：', len(m.list()[1]))
m.quit()
"
```

**问题：容器重启后代码回退**
- 说明是用 `docker cp` 更新的代码，没有打进镜像
- 需要重新 build 镜像并部署（见第五节 5.1）
