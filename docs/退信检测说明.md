# 退信检测功能说明

## 一、什么是退信检测

系统批量发完邮件后，可以主动检查哪些邮件被对方服务器拒收了。
被拒收的邮件叫"退信"，比如收件人邮箱不存在、域名不存在、对方服务器拒绝等情况。

---

## 二、退信是怎么产生的

```
我们发邮件 → 对方邮件服务器收到后拒绝 → 对方服务器自动发一封"退信通知"给我们的发件邮箱
```

退信通知是一封普通邮件，发件人是对方服务器（一般是 postmaster@ 或 mailer-daemon@），
内容里会写明哪个邮箱地址无法送达、原因是什么。

---

## 三、之前用 QQ 邮箱为什么能做

用 QQ 邮箱（464197787@qq.com）发件时：

```
系统发件 → QQ SMTP 服务器 → 对方服务器拒绝 → 退信通知发回 QQ 收件箱
```

QQ 邮箱的收件箱是完整的，退信通知会正常回到这个账号的收件箱里。
我们用 IMAP/POP3 连接 QQ 收件箱，能读到这些退信通知，从里面提取出被退信的邮箱地址。

**所以能做的关键：发件账号的收件箱能正常收到退信通知。**

---

## 四、换成企业邮箱后为什么不能做

企业邮件系统架构（张老师白板图）：

```
我们的系统
    ↓ 发件
192.168.32.8（邮件网关）
    ↓ 转发
180.168.32.10（内部邮件服务器）
```

问题出在这里：当对方服务器发退信通知回来时，通知发给了 192.168.32.8，
但 32.8 并不会把退信通知转存到 filingmail@message.sse.net.cn 这个账号的收件箱里。

也就是说：

```
对方服务器拒绝 → 退信通知发回 192.168.32.8 → 通知在这里丢失，没有进 filingmail 的收件箱
```

我们连上 filingmail 的 POP3 收件箱，里面根本没有退信通知，自然检测不到。

---

## 五、解决方案

需要张老师在邮件服务器上做一个配置：

**把退信通知（Bounce/DSN）转发到 filingmail@message.sse.net.cn 的收件箱。**

这在 Dovecot（他们用的邮件服务器）里是标准配置，操作并不复杂，
具体由运维人员在服务器上配置 Return-Path 或 DSN 转发规则即可。

配置完成后，我们的代码不需要任何改动，退信检测功能即可正常使用。
